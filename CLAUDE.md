# Rutgers Physics & Astronomy — Department Signage

Digital signage for two TVs at the Physics & Astronomy department entrance,
driven by a Raspberry Pi running Chromium in kiosk mode.

## Repository layout

```
physics-signage/
├── update.py            # TV1: scrapes events/news, generates output/display.html
├── setup.sh             # TV1: one-time Pi setup (venv, cron, kiosk autostart)
├── requirements.txt     # Python deps for TV1 (requests, bs4, qrcode)
├── output/              # Generated by update.py — not in git
│   ├── display.html
│   ├── events.json
│   └── irons_poster.jpg
├── tv2/
│   ├── generate_slideshow.py  # TV2: scans media/, generates tv2/index.html
│   ├── sync_gdrive.sh         # TV2: rclone sync from Drive + regenerate
│   ├── setup_tv2.sh           # TV2: one-time Pi setup
│   └── media/
│       ├── captions.json      # Per-file title + description (optional)
│       └── *.jpg / *.mp4 …    # Media files (not in git)
└── .github/workflows/update.yml   # GitHub Actions: hourly build → GitHub Pages
```

## TV1 — Events & News display

**Script:** `update.py`  
**Output:** `output/display.html`

### Data sources

| Data | Source | Method |
|------|--------|--------|
| Colloquium + seminars | physics.rutgers.edu/newsletter | `requests` + `BeautifulSoup` |
| Colloquium abstracts | physics.rutgers.edu/events/colloquium | Playwright (optional) |
| Irons Public Lecture | physics.rutgers.edu/irons | `requests` + `BeautifulSoup` |
| Department news | physics.rutgers.edu/news/2026-news | `requests` + `BeautifulSoup` |

### Key implementation details

- **Newsletter parser** (`_parse_newsletter`): state machine over a `<pre>` block.
  Two-line event format: type line (`CMS  Tues Apr 7 \t Speaker, Institution`)
  followed by a detail line (`1:30   330W and Zoom   "Title"`).
  Regexes: `_TYPE_LINE`, `_DETAIL_LINE`, `_LINK_LINE`.

- **Bot-challenge bypass**: physics.rutgers.edu uses Cloudflare slider challenges.
  Playwright (`_solve_slider_challenge`) simulates a human mouse drag.
  Newsletter scraping does NOT require Playwright.

- **Irons lecture**: all event data lives in one `<img>` alt attribute.
  Parsed with `r'\bby\s+(.+?)\s+of\s+(.+?),\s+held\b'`.

- **QR codes**: generated as base64 SVG (no Pillow needed) via `qrcode` library.
  One code per slide linking to the relevant department page.

- **News ticker**: bottom of display, news items only (no colloquium).
  Font size `1.25rem`, color `#dde4ff`, red "Updates" badge.

- **Slide order**: this-week colloquium → Irons lecture → day seminars (Mon–Fri)
  → department news → next-week colloquium → next-week day seminars.

### Update cycle

- Pi: hourly cron runs `venv/bin/python update.py`
- GitHub Pages: hourly GitHub Actions workflow, also triggers on push

## TV2 — Research Highlights slideshow

**Script:** `tv2/generate_slideshow.py`  
**Output:** `tv2/index.html`

### Adding content

1. Drop `.jpg`, `.png`, `.gif`, `.mp4`, `.webm` files into `tv2/media/`.
2. Prefix filenames with `01_`, `02_` … to control display order.
3. Optionally add entries to `tv2/media/captions.json`:
   ```json
   {
     "01_dark_matter.jpg": {
       "title": "Dark Matter Search",
       "description": "Prof. Smith's group at Rutgers"
     }
   }
   ```
4. Run `python3 tv2/generate_slideshow.py` (or let the cron do it).

### Google Drive sync

Media is submitted by faculty via a shared Google Drive folder named
**`Physics Signage Media`**. rclone pulls from `gdrive:Physics Signage Media`.

```bash
# Manual sync
bash ~/physics-signage/tv2/sync_gdrive.sh

# Configure rclone (one time)
rclone config   # name the remote "gdrive", type: Google Drive
```

### Display design

- Full-screen black background, `object-fit: contain`
- **Top banner**: red gradient (`rgba(180,0,30,…)`), atom icon ⚛,
  "Rutgers Physics & Astronomy / Research Highlights"
- **Bottom caption**: dark gradient overlay, title + description
- **Progress bar**: red, animates across image duration (default 12 s)
- **Videos**: advance on `onended`; 3-minute safety timeout
- **Auto-reload**: every 30 minutes to pick up newly synced media

## Raspberry Pi deployment

### TV1

```bash
git clone https://github.com/b0burk02/physics-signage.git ~/physics-signage
cd ~/physics-signage
bash setup.sh
sudo reboot
```

### TV2

```bash
# After cloning (same repo)
bash ~/physics-signage/tv2/setup_tv2.sh
rclone config   # if not already done
bash ~/physics-signage/tv2/sync_gdrive.sh
sudo reboot
```

### Two-monitor Pi config

Add to `/boot/firmware/config.txt`:
```
hdmi_force_hotplug:0=1
hdmi_force_hotplug:1=1
```

Add to `~/.config/lxsession/LXDE-pi/autostart` (before Chromium lines):
```
@xrandr --output HDMI-1 --same-as HDMI-2
```
